# 웹툰 관심수 추적 대시보드

titleId **851689** 작품의 관심수를 매시간 자동으로 기록하고, 웹사이트에서 추이를 확인할 수 있는 프로젝트입니다.

## 구성

| 파일 | 역할 |
|---|---|
| `collect.py` | 네이버 웹툰에서 관심수를 가져와 `data/history.csv`에 기록 |
| `.github/workflows/collect.yml` | 매시간 자동 실행 설정 (매시 7분) |
| `index.html` | 추이 그래프·시간별 증가량·일별 표를 보여주는 대시보드 |
| `data/` | 수집된 데이터가 쌓이는 폴더 (자동 생성) |

## 세팅 순서 (약 10분)

### 1. GitHub 저장소 만들기

1. github.com 로그인 → 우측 상단 **+** → **New repository**
2. 이름은 자유롭게 (예: `webtoon-tracker`)
3. **Public** 선택 — 무료 계정에서 GitHub Pages를 쓰려면 Public이어야 합니다
4. **Create repository**

> ⚠️ Public 저장소는 데이터(관심수 기록)가 누구나 볼 수 있는 상태가 됩니다.
> 비공개로 하고 싶다면 Private 저장소 + 유료 플랜(Pages 지원)이 필요하거나,
> 대시보드 없이 데이터 수집만 Private로 돌리는 방법도 있습니다.

### 2. 파일 업로드

가장 쉬운 방법은 웹에서 드래그 앤 드롭:

1. 저장소 페이지에서 **uploading an existing file** 링크 클릭
2. 이 압축을 푼 파일들을 전부 끌어다 놓기
3. **Commit changes**

> ⚠️ 중요: `.github` 폴더는 숨김 폴더라서 드래그할 때 빠지기 쉽습니다.
> 업로드 후 저장소에 `.github/workflows/collect.yml`이 보이는지 꼭 확인하세요.
> 웹 업로드로 폴더가 안 들어가면: 저장소에서 **Add file → Create new file** →
> 파일명에 `.github/workflows/collect.yml` 입력(자동으로 폴더 생성됨) →
> 내용 붙여넣기 → Commit.

### 3. 첫 수집 테스트 (수동 실행)

1. 저장소 상단 **Actions** 탭 클릭
2. 처음이면 "워크플로 활성화" 확인 버튼이 나올 수 있음 → 활성화
3. 왼쪽에서 **관심수 수집** 선택 → 우측 **Run workflow** → **Run workflow**
4. 1~2분 뒤 초록 체크가 뜨면 성공. `data/history.csv`가 생겼는지 확인
5. ❌ 빨간 X가 뜨면 로그를 열어 `[FAIL]` 메시지를 복사해서 Claude에게 보여주세요 —
   API 응답 형식이 바뀐 경우 스크립트를 수정하면 됩니다

### 4. GitHub Pages 켜기 (대시보드 배포)

1. 저장소 **Settings** → 왼쪽 메뉴 **Pages**
2. Source: **Deploy from a branch**
3. Branch: **main** / 폴더 **/ (root)** → **Save**
4. 1~2분 뒤 상단에 주소가 표시됩니다:
   `https://아이디.github.io/저장소이름/`

이 주소가 대시보드입니다. 북마크해 두고 아무 때나 열어보세요. 폰에서도 잘 보입니다.

### 5. 끝!

이후로는 매시간 자동으로 기록됩니다. 별도 관리가 필요 없습니다.

## 알아두면 좋은 것

- **실행 시각이 정확히 정각이 아닙니다.** GitHub의 예약 실행은 몇 분에서 수십 분까지
  밀릴 수 있습니다(무료 인프라 특성). 추이 파악에는 영향이 없습니다.
- **60일 규칙**: 저장소에 오랫동안 활동이 없으면 GitHub이 예약 실행을 자동 중지시킬 수
  있습니다. 중지 전 이메일이 오고, Actions 탭에서 버튼 한 번으로 재개됩니다.
- **수집 주기 변경**: `collect.yml`의 `cron: "7 * * * *"`을 수정하면 됩니다.
  예: 30분마다 → `"7,37 * * * *"` (단, 너무 촘촘하면 무료 사용량을 빨리 소모합니다)
- **다른 작품 추가**: `collect.py`의 `TITLE_ID`를 바꾸거나, 여러 작품을 추적하고 싶으면
  Claude에게 다중 작품 지원 버전을 요청하세요.
- **데이터 백업**: 모든 기록이 git 히스토리에 남으므로 저장소 자체가 백업입니다.
  `data/history.csv`를 내려받아 엑셀에서 열 수도 있습니다.
